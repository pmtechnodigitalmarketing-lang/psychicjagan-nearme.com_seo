Critical Code Bugs

BUG #1 — Duplicate HTML Document Structure (SEVERITY: CRITICAL) header.php outputs a complete <!DOCTYPE html><html><head></head><body> structure. index.php also outputs its own complete structure and then include('header.php'). Result: Google receives nested HTML documents. This invalidates all schema, canonical, and meta signals on the homepage. 
// index.php line 20:
<!DOCTYPE html><html lang="en"><head>...</head><body>
// Then line 168:
<?php include 'header.php'; ?>
// header.php ALSO starts with:
<!doctype html><html lang="en"><head>...</head><body>
BUG #2 — Image Hotlinking from Pinterest CDN (SEVERITY: CRITICAL) The astrologer's primary profile image is hosted at https://i.pinimg.com/736x/71/44/22/714422543e98763d0f185715fc458124.jpg. This is a random Pinterest image — not a photo of Psychic Jagan Nath. Google's vision systems will confirm this person is not who the site claims. This is a direct E-E-A-T and spam policy red flag.
BUG #3 — EMPTY Privacy Policy & Terms (SEVERITY: HIGH) privacy-policy.php and terms-conditions.php are 0-byte files. Rendering these URLs returns a blank page. Google Quality Raters flag missing legal pages as trust failures on YMYL sites.
BUG #4 — Address Data: ALL OVER USA (SEVERITY: HIGH) astrologer-data.php sets 'street' => '', 'state' => '', 'postal' => '', 'city' => 'ALL OVER USA'. This propagates into all LocalBusiness schema and footer NAP. Google will not confirm a physical location, blocking Local Pack eligibility entirely.
BUG #5 — Google Business Profile Field Empty (SEVERITY: HIGH) 'google_business_profile' => ''. No GBP link exists in the data file. Without a verified GBP, this business cannot rank in Google Maps or the Local Pack for any psychic/astrologer query.
BUG #6 — sameAs Array Contains Only Empty String (SEVERITY: HIGH) 'sameAs' => ['']. The JSON-LD sameAs property references zero real social profiles. Google uses sameAs to confirm entity identity. An empty array removes all entity verification.
BUG #7 — State Pages Missing Schema (SEVERITY: MEDIUM) services.php (which serves all 50 state pages) has no @type: LocalBusiness, FAQPage, or BreadcrumbList schema. Only the FAQ items are defined in PHP variables but never structured as JSON-LD. Lost rich result opportunity for 50 pages.
BUG #8 — Canonical Mismatch on Service Pages (SEVERITY: MEDIUM) Service pages output canonical as /love-reading.php but .htaccess rewrites clean URLs. If Google indexes /love-reading (no .php) and canonical says /love-reading.php, signal dilution occurs. All service page canonicals should point to the clean URL.
BUG #9 — Footer Links Use Relative URLs (SEVERITY: MEDIUM) Footer outputs href="black-magic-removal.php" — relative paths. On state pages served at /services-in-colorado, these may resolve incorrectly depending on server config, creating potential 404 signals.
BUG #10 — Logo is Local PNG, 1.5MB (SEVERITY: MEDIUM) images/logo.png is 1.48MB. The hero images are 2.2–2.7MB each. These massive uncompressed PNGs will catastrophically fail Core Web Vitals LCP targets on mobile.
